    PROJECT_C Department 3 Sass documentation

    SassDoc is a documentation system specifically for Sass functions, mixins and variables. It goes beyond the living style guide (which displays rendered elements) by documenting options, attributes, and dependencies of Sass functionality.

    Specific to this department theme, this documentation inherits any Sass elements and documentation from the base theme.

    Like KSS, it is generated automatically as part of the build process (gulp sassdoc) and is derived from appropriate annotations within the Sass file.

    Navigate this documentation using the groups at left or by searching above.

    Responsive breakpoints

    variables

    mq-breakpoints

    $mq-breakpoints: (
  small: 320px,
  medium: 769px,
  large: 1000px,

  // Tweakpoints
  // desktopAd: 810px,
  // mobileLandscape: 480px
);

    Description

    Breakpoints are used mobile-first; that is, putting something inside a "medium" breakpoint means that it will be applied from the medium breakpoint and above. Styles meant for screen widths smaller than the "small" breakpoint should simply be set outside of a breakpoint wrapper, and then overridden if need be.

    Also see the max-content $widths variable in _sizes.scss.

    Example

    Sass

    p {
  display: block;

  @include mq(medium) {
    display: inline;
  }
}

@include mq(medium) {
  div {
    display: inline;
  }
}

    CSS output

    p {
  display: block;
}

@media (min-width: 48.0625em) {
  p {
    display: inline;
  }
}

@media (min-width: 48.0625em) {
  div {
    display: inline;
  }
}

    Used by

    • [variable] mq-static-breakpoint

    Links

    • https://github.com/sass-mq/sass-mq
    • https://sass-mq.github.io/sass-mq

    Author

    • Sass MQ

    mq-responsive

    $mq-responsive: true;

    Description

    To enable support for browsers that do not support @media queries, (IE <= 8, Firefox <= 3, Opera <= 9) set $mq-responsive to false Create a separate stylesheet served exclusively to these browsers, meaning @media queries will be rasterized, relying on the cascade itself

    mq-static-breakpoint

    $mq-static-breakpoint: large;

    Description

    Define the breakpoint from the $mq-breakpoints list that should be used as the target width when outputting a static stylesheet (i.e. when $mq-responsive is set to 'false').

    Requires

    • [variable] mq-breakpoints

    mq-show-breakpoints

    $mq-show-breakpoints: ();

    Description

    If you want to display the currently active breakpoint in the top right corner of your site during development, add the breakpoints to this list, ordered by width, e.g. (mobile, tablet, desktop). Turned off in production.
